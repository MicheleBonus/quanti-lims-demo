# DB-Migration: SQLite → PostgreSQL + Alembic

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the 200-line manual `migrate_schema()` function and SQLite with PostgreSQL + Flask-Migrate (Alembic), so schema changes are managed by generated migration files instead of hand-written PRAGMA SQL.

**Architecture:** Flask-Migrate registers `flask db` CLI commands that wrap Alembic. Tests continue to use SQLite in-memory via `db.create_all()` (guarded by `TESTING` flag). Production uses PostgreSQL via Docker Compose and `flask db upgrade` on deploy. The ORM models in `models.py` are unchanged.

**Tech Stack:** Flask-Migrate 4.x, psycopg2-binary 2.9.x, PostgreSQL 16 (Docker), Docker Compose v2

**Spec:** `docs/superpowers/specs/2026-03-18-ux-redesign-two-phase-design.md` (section "Technische Änderungen")

Update `create_app()`:

```python
def create_app(test_config: dict | None = None):
    app = Flask(__name__)
    app.config.from_object(Config)
    if test_config is not None:
        app.config.update(test_config)
    CSRFProtect(app)
    db.init_app(app)
    Migrate(app, db)  # registers `flask db` CLI commands

    with app.app_context():
        if app.config.get("TESTING"):
            db.create_all()   # tests: create schema directly (SQLite in-memory)
        from init_db import seed_database
        seed_database()

    register_routes(app)
    register_filters(app)
    register_error_handlers(app)
    return app
```

## How to add future schema changes

When a model in `models.py` is changed (new column, new table, etc.):

```bash
flask db migrate -m "describe what changed"
flask db upgrade
```

That's it. No PRAGMA queries, no manual ALTER TABLE.
